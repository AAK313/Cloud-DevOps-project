<details>

## ğŸ› **Troubleshooting Guide**

</div>

<details>
<summary><b>ğŸ”´ Terraform Backend Errors</b></summary>

**Symptom:** State locking failures or access denied

**Solutions:**
```bash

# Verify IAM permissions

```

</details>

<details>
<summary><b>ğŸ”´ EKS Access Issues</b></summary>

**Symptom:** Cannot connect to cluster or kubectl commands fail

**Solutions:**
```bash

# Verify AWS credentials
aws sts get-caller-identity

# Check cluster status
aws eks describe-cluster --name devops-eks-cluster

# Verify aws-auth ConfigMap
kubectl get configmap aws-auth -n kube-system -o yaml
```

</details>

<details>
<summary><b>ğŸ”´ GitHub Actions Pipeline Failures</b></summary>

**Symptom:** CI/CD workflow fails or secrets not found

**Solutions:**
- Verify all required secrets are set in repository settings
- Use Docker Hub Personal Access Token, not password
- Check workflow logs for specific error messages
- Test locally with `act` to debug workflows
- Ensure branch protection rules aren't blocking commits

</details>

<details>
<summary><b>ğŸ”´ ArgoCD Sync Issues</b></summary>

```bash
# Restart ArgoCD if needed
kubectl rollout restart -n argocd deployment/argocd-server
```

</details>

<details>
